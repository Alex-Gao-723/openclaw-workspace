## 4️⃣ Claude 产品矩阵对比

> 来源：Shraddha Bharuka (@BharukaShraddha)

### 三种形态对比

| 产品 | 定位 | 适用场景 | 优点 | 缺点 |
|------|------|----------|------|------|
| **Claude AI** | 对话式AI助手 | 写邮件、文章、头脑风暴、研究 | 零设置、通用性强 | 无系统访问、需手动跟进 |
| **Claude Code** | 命令行编程Agent | 多文件开发、重构代码库、自动测试修复 | 完全自主、深度代码上下文 | 需仔细审查、有使用成本 |
| **Claude Cowork** | 桌面端非开发者工具 | 批量文件管理、PDF数据提取、跨应用自动化 | 无需编码、跨应用工作流 | Beta阶段、配置自由度低 |

### 关键洞察

**Claude Code** - 最激进的定位：
- 直接在终端运行，真正自主的编程 Agent
- 不仅回答问题，而是直接写/改/测试代码
- 与 Cursor/Copilot 不同，命令行驱动

**Claude Cowork** - 瞄准非开发者：
- 类似 OpenClaw 的愿景（普通人用AI自动化）
- 批量文件操作、数据提取、跨应用流程

### 工作流对比
- **Claude AI**: Prompt → Chat UI → Claude Models → Output (Limited Memory/Tools)
- **Claude Code**: Prompt → CLI UI → File-system Memory + Multi-File editing → Write/Build
- **Claude Cowork**: Prompt → Chat UI → Organize files/Edit file/Multi-cross app workflows → Output
